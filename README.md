# Golden Dataset for Evaluating RAG

A reproducible pipeline for building citation-grounded evaluation datasets to benchmark Retrieval-Augmented Generation (RAG) systems.

This project ingests educational YouTube transcripts, cleans and summarizes them, generates candidate question–answer pairs, and curates a golden dataset with timestamped source citations. The result is a fixed benchmark for measuring retrieval accuracy, answer faithfulness, and end-to-end RAG performance.

---

## Project Overview

RAG systems combine retrieval (finding relevant passages) with generation (synthesizing answers from context). Evaluating them requires known questions, reference answers, and traceable citations back to source material.

This repository provides:

- A **reproducible data pipeline** from YouTube captions to structured QA pairs
- A **candidate pool** of 20 grounded questions across four ML/DL videos
- A **golden dataset** of five curated questions selected for retrieval difficulty and diversity
- **Technical reports** documenting methodology and future RAG integration

The corpus spans English and Hindi instructional content from 3Blue1Brown, CampusX, and CodeWithHarry, covering neural networks, transformers, and deep learning.

## Installation

### Prerequisites

- Python 3.10+
- An [Anthropic API key](https://console.anthropic.com/settings/keys) (required for summarization and question generation)

### Setup

```bash
git clone https://github.com/adarshm0602/Golden-Dataset-for-Evaluating-RAG-.git
cd Golden-Dataset-for-Evaluating-RAG-

python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

pip install -r requirements.txt
```

### Configuration

Copy the environment template and add your API key:

```bash
cp .env.example .env
```

Edit `.env`:

```env
ANTHROPIC_API_KEY=your-api-key-here
ANTHROPIC_MODEL=claude-sonnet-4-6
ANTHROPIC_MAX_TOKENS=4096
```

Add or modify source videos in `videos.json`:

```json
{
  "title": "But what is a Neural Network?",
  "channel": "3Blue1Brown",
  "url": "https://www.youtube.com/watch?v=aircAruvnKk",
  "languages": ["en"]
}
```

## Usage

Run the pipeline scripts in order. Each script reads from the previous stage's output directory.

### 1. Download transcripts

```bash
python3 scripts/download_transcripts.py
```

Fetches YouTube captions for every video in `videos.json` and writes timestamped JSON and TXT files to `data/raw_transcripts/`.

### 2. Clean transcripts

```bash
python3 scripts/clean_transcripts.py
```

Deduplicates subtitle fragments, merges them into coherent paragraphs, and writes cleaned files to `data/cleaned_transcripts/`.

### 3. Generate summaries

```bash
python3 scripts/summarize.py
```

Calls Claude to extract executive summaries, topics, concepts, definitions, and examples. Output: one Markdown file per video in `data/summaries/`.

### 4. Generate candidate questions

```bash
python3 scripts/generate_candidate_questions.py
```

Reads summaries and cleaned transcripts, produces approximately 20 candidate QA pairs, and writes `candidate_questions.csv`.

### 5. Select golden questions

```bash
python3 scripts/select_golden_questions.py
```

Evaluates all candidates against quality and diversity criteria, selects the best five, and writes `golden_dataset.csv`.

### 6. Run retrieval benchmark

```bash
python3 scripts/run_retrieval_benchmark.py
```

Indexes paragraph chunks from `data/cleaned_transcripts/`, embeds them with a multilingual sentence-transformer, and scores retrieval against both `candidate_questions.csv` (20 rows) and `golden_dataset.csv` (5 rows). Ground truth: a hit is correct when the retrieved chunk's video title matches and the cited timestamp falls within the chunk's `[start, end]` range.

Writes results to `reports/benchmark_results.json` and `reports/benchmark_results.md`.

### Source corpus

### Candidate questions (`candidate_questions.csv`)

20 question–answer pairs generated across all four videos with varied difficulty (Easy, Medium, Hard) and topics. Each row includes:

| Column | Description |
|--------|-------------|
| `Question` | Natural-language query |
| `Answer` | Grounded reference answer |
| `Video` | Source video title |
| `Timestamp` | Start time of supporting passage (`HH:MM:SS`) |
| `Difficulty` | Easy / Medium / Hard |
| `Topic` | Subject label |
| `Why this question is useful for evaluating retrieval` | Retrieval benchmarking rationale |

### Golden dataset (`golden_dataset.csv`)

Five questions selected for maximum evaluation value:

| Video | Topic | Difficulty |
|-------|-------|------------|
| All About ML & Deep Learning | Reinforcement Learning | Medium |
| What is Deep Learning? | Interpretability (black box) | Hard |
| Transformers, the tech behind LLMs | Word embedding semantics | Hard |
| But what is a Neural Network? | Bias in neural networks | Medium |
| What is Deep Learning? | Deep Learning success factors | Hard |

The golden set includes all candidate columns plus `Selection Rationale` explaining why each question was chosen.

## Benchmark Results

Retrieval benchmark run on **123 paragraph chunks** from four cleaned transcripts, using `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2`. Reproduce with `python3 scripts/run_retrieval_benchmark.py`.

### Overall

| Dataset | Questions | Hit@1 | Hit@3 | Hit@5 | MRR |
|---------|-----------|-------|-------|-------|-----|
| Candidate questions | 20 | 15.0% | 15.0% | 30.0% | 0.185 |
| Golden dataset | 5 | 20.0% | 20.0% | 40.0% | 0.250 |

### Candidate set — by query type

| Medium | 9 | 11.1% | 11.1% | 11.1% | 0.111 |

## Methodology

The full pipeline is documented in [`reports/methodology.md`](reports/methodology.md). Key stages:

1. **Transcript collection** — YouTube captions fetched via `youtube-transcript-api` with language preferences from `videos.json`.
2. **Cleaning** — Whitespace normalization, duplicate removal, and paragraph segmentation using pause and length heuristics.
3. **Summarization** — Claude extracts structured summaries (topics, concepts, definitions, examples) grounded strictly in transcript text.
4. **Candidate generation** — Claude produces ~20 diverse QA pairs with timestamps, difficulty labels, and retrieval rationale.
5. **Golden selection** — Claude selects the best five questions based on video coverage, concept diversity, difficulty, specificity, and answer quality.

Selection criteria prioritize questions that would fail if retrieval returns the wrong video, an adjacent topic, or a generic web passage.
